## 一言で言うと

**スキルは呼ばれて初めて動く。自動適用されない。CLAUDE.md に書くのが一番確実。**

## 背景

著者がプロジェクト固有のコーディング規約をスキルとして定義したが、Claude に自動で守らせようとしたところ全く機能しなかった。その原因を実験で可視化した記事。

## 主な発見

### 1. Skill ツールはデフォルトでブロックされている

スキルを作成しても、`Skill` ツール自体の実行権限がデフォルトで拒否されている。

**解決策**: `.claude/settings.json` に許可を追加する。

```json
{
  "permissions": {
    "allow": ["Skill(*)"]
  }
}
```

### 2. スキルは「暗黙適用」されない

スキルは明示的に呼び出されて初めて有効になる。コンテキストとして自動適用されるわけではない。

### 3. カスタムサブエージェントの発動率が低い (約25%)

カスタムサブエージェントの `description` に記述したトリガー語でしか発動しない。公式エージェント（`claude-code-guide` など）はシステムプロンプトに発動条件がハードコードされているため発動率が高い。

### 4. 最も確実な方法: CLAUDE.md に記述する

CLAUDE.md にエージェントの起動条件を明記することで、**発動率が100%** になる。

```markdown
## line-counter エージェント

ユーザーが以下のいずれかを尋ねた場合、必ず Task ツールで
`line-counter` エージェント（subagent_type='line-counter'）を使用すること：

- ファイルの行数について
- 行のカウントについて
```

## 機能の使い分け

| 機能 | 自動適用 | 用途 |
|------|---------|------|
| CLAUDE.md | ✅ | ルール・制約・エージェント起動条件 |
| サブエージェント + CLAUDE.md | ✅ | 厳密なタスク実行 |
| スキル単体 | ❌ | エージェント間の共有知識ベース |

## 推奨ディレクトリ構成

```
CLAUDE.md                （全体ルールとエージェント起動条件）
.claude/agents/          （特定タスク用サブエージェント）
.claude/skills/          （共有知識・オプション）
```

## 公式ドキュメント未記載の5点

1. **Skill ツールはデフォルト権限拒否** — 明示的に許可が必要
2. **スキルは暗黙適用されない** — 呼び出しがあって初めて機能する
3. **カスタムサブエージェントの発動率は低い** — description のトリガー語依存
4. **CLAUDE.md が最も確実** — 起動条件を書けば発動率100%
5. **スキルはエージェント間で共有できる** — 複数サブエージェントが同じ知識を参照可能

## コメント

この記事の知見は自分の CLAUDE.md 設計にも直接使える。スキルに「この条件で使え」と書いても Claude が勝手に読んでくれるわけではなく、CLAUDE.md で「このスキルをこの条件で呼べ」と指示するのが正解というのは盲点だった。

スキルは「ライブラリ」、CLAUDE.md は「エントリポイント」と考えると整理しやすい。
