# Godot でのヴォルティング実装

**参考動画：** [Vaulting in Godot 4 - El3ctroNam Studios](https://www.youtube.com/watch?v=aRDsURbUk5w)

## 概念

プレイヤーがジャンプ中に壁や障害物に接触したときに自動的に登り越す「ヴォルティング」機能。単純な衝突判定では壁をすり抜けてしまうため、複数のレイキャストを組み合わせた判定システムが必要。

## 実装の基本原理

### レイキャスト判定システム

2種類のレイキャスト配列を使用：

1. **ChestRay**（1本）
   - プレイヤーの胸部の高さから前方へ発射
   - 壁や障害物に接触しているかを検出
   - 登り判定のトリガーになる

2. **HeadRay配列**（3本 - center, left, right）
   - プレイヤーの頭上から前方へ発射
   - 登り越せる十分な高さがあるか確認
   - **いずれかが衝突 → 登ることができない**
   - **すべて非衝突 → 登ることができる**

### 登り判定の流れ

```
ジャンプ + Jump入力
  ↓
ChestRayが衝突？ → No → ジャンプ終了
  ↓ Yes
HeadRayすべてが非衝突？ → No → ジャンプ終了
  ↓ Yes
→ 登り処理を実行
```

## 実装のポイント

### 制御フラグ

- `canJump`：登り中はfalse（登り終わるまでジャンプ禁止）
- `canMove`：カメラと移動を制限
- `isCrouching`：しゃがみ状態の判定

### アニメーション順序

登り処理は複数のステップに分かれて実行：

1. **垂直移動フェーズ**（0.4秒）
   - プレイヤーの位置をY軸で上に移動
   - カメラを上向きに回転（X軸で-20度程度）
   - カメラにZ軸でランダムな微かな揺らぎを追加

2. **水平移動フェーズ**（垂直完了後、0.2秒）
   - 垂直移動が完了するまで待機（await）
   - プレイヤーを前方へ移動（Z軸で1.2倍）
   - カメラの回転をリセット

### Tween による実装

Godot の `Tween` システムで滑らかなアニメーションを実現：

```
Tween.set_trans(Tween.TRANS_CUBIC)  # キュービック補間
Tween.set_ease(Tween.EASE_IN)      # イージング適用
```

- 直線的でなく、自然な加速度を与える
- 複数のアニメーションを並列・順序制御可能

## 設計の工夫

- **パラメータの分離**：値をコードの冒頭で定義し、調整しやすくする
- **条件分岐による柔軟性**：しゃがみ中と立った状態で異なるアニメーション
- **複数レイキャストの配列化**：ループで一括判定が可能
