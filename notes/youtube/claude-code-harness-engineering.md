## ハーネスエンジニアリングとは

「ハーネス」は馬をコントロールするための馬具のこと。Claude Code をハーネスで制御するように、**人間が手を加えずにAIが自律的にアプリ開発を行う**ための設計思想。

AIエンジニアリングの進化：
> プロンプトエンジニアリング → コンテキストエンジニアリング → **ハーネスエンジニアリング**

人間がゴール設定をすれば、AIがそのゴールまで改善を繰り返す。最終成果物を人間がレビューするだけで良い状態を目指す。

---

## 3つのサブエージェント構成

```
人間のざっくり指示
       ↓
  [プランナー] ── 仕様書・要件定義を自動生成
       ↓
  [ジェネレーター] ── 仕様書に基づいて1機能ずつ実装
       ↓
  [エバリューエーター] ── Playwright MCPでブラウザ操作してテスト・レビュー
       ↓           ↑
    不合格なら差し戻し（自己改善ループ）
       ↓
    合格 → 次の機能へ
```

| エージェント | 役割 | 人間界での対応 |
|---|---|---|
| プランナー | 仕様書生成 | PM / テックリード |
| ジェネレーター | 実装 | フルスタックエンジニア |
| エバリューエーター | テスト・レビュー | QA / レビュアー |

- 全エージェントに **Claude Opus モデル** を使用
- サブエージェントにはスキルやMCPを個別付与できる
- エバリューエーターに **Playwright MCP** を付与してブラウザ自動テストを実現

## 実演：AI旅行プランナーアプリを自律開発

### セットアップ手順

1. Cursor で新規プロジェクト作成
2. 3つのサブエージェント（`/.claude/agents/`配下）をClaudeに自動作成させる
3. Playwright MCPをエバリューエーターに設定（「設定して」と頼むだけ）
4. `CLAUDE.md` でオーケストレーションのルールを定義
5. `--dangerously-skip-permissions` で人間の許可なしに自動実行

### 渡した指示（ざっくりで十分）

> 「旅行プランナーアプリを作りたい。DB・認証不要。Claude APIを使って都道府県・都市を指定したら最適な旅行プランを提示するWebアプリ。デザインは旅行アプリに適したものを採用して。」

### 結果

- 約1時間でスプリント3まで完成
- Playwright MCPがブラウザを自動起動してレスポンシブ確認まで実施
- 広島旅行プランを実際に生成できるWebアプリが完成（Claude Sonnet使用）

## ポイント・Tips

- **品質向上が顕著**: フィードバックループを1回回すだけでデザイン・機能品質が大きく向上
- **複雑なアプリでも2〜3時間**で一定クオリティのものが完成
- **コスト**: 1時間で約59,000トークン消費 → 200ドルプランを推奨
- 並行実行したい場合は **Claude Agent Teams** を活用
- デザインを高めるには **UIデザイナー専用エージェント** や **AIデザイナーMCPサーバー** を追加する
- 参考記事・GitHubリポジトリが概要欄に掲載されている
