# 世界で最も信頼されるコードが Rust で書き直されている

**動画:** The most trusted code on Earth is being rewritten in Rust
**チャンネル:** Fireship
**日付:** 2026-06-19

## SQLite の歴史

- **2000年開発開始** — 海軍のミサイル駆逐艦制御システム開発者 D. Richard Hipp により開発
- **設計の革新性** — データベースサーバーが不要で、アプリケーションに直接埋め込める
- **現在の利用状況** — 地球上の数兆個のデバイスに搭載（iPhone、Android、Windows、Mac、ブラウザ、火星ローバーなど）
- **信頼性の基盤** — C で実装され、膨大なテストスイートにより25年かけて信頼を獲得

## SQLite の課題

- ソースコードは公開されているが、**オープンソース開発ではない**
- メンテナンスは Richard Hipp と2人の開発者のみで行われている
- **外部からの貢献を受け付けていない** — 信頼性は保つが、革新は遅い
- 新機能追加の自由度がない

## Turso — Rust による SQLite の書き直し

**開発者の背景：**
- 遅延（Latency）に関する本の著者
- Linux カーネルの top-5 コントリビューター（Linus 認定）

### 主な新機能

| 機能 | SQLite | Turso |
|------|--------|-------|
| **並行性** | 1 つのライターのみ | 複数のライターが同時アクセス可（異なる行なら競合なし） |
| **非同期処理** | ディスク操作でスレッド ブロック | 操作を手放し、他のタスク実行可能 |
| **ベクトル検索** | 別の vector DB が必要 | ネイティブ サポート（AI 埋め込み用） |
| **後方互換性** | N/A | SQLite ファイルとの完全互換 |

### ベクトル検索の意義

- AI アプリには **埋め込みデータの保存・検索** が必須
- 通常は Pinecone など別の vector DB を使う → 複雑性 2 倍
- **Turso は同じファイル内に統合** — 25年前に Hipp が SQLite でやった「必要な機能をライブラリに詰め込む」トリックを繰り返す

## 信頼性の確保 — 確定的シミュレーション

Turso は「**Deterministic Simulation**」というテスト戦略を採用：

1. データベース全体をシミュレートされた環境で実行
2. 時間、ネットワーク、ディスク挙動を完全に制御
3. 障害をシミュレート
   - 電源喪失
   - 途中でのライト失敗
   - ディスク破損
   - ディスクが「保存した」と嘘をつく
4. **同じランダムシードで再現** — バグを確実に修正

## オープンソース化

- Turso は **真のオープンソース** — 外部コントリビューション受け入れ
- SQLite のコードは Public Domain（パブリックドメイン）なので、参考にして Rust で実装
- SQLite の哲学「シンプルさと信頼性」を保ちながら、現代的機能を追加

## 評価

> "Probably not a good idea, but you're only viewed as an idiot until it works."
> — Fireship

SQLite の書き直しは狂気に見えるが、以下の点で価値がある：
- **真のオープンソース化** — 外部からの改善が可能
- **現代的ワークロード対応** — 並行性、非同期、ベクトル検索
- **Drop-in replacement** — 既存コードを書き直さずに移行可能
