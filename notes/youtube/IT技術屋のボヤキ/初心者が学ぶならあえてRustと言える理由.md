# 初心者が学ぶならあえてRustと言える理由

**チャンネル**：IT技術屋のボヤキ  
**URL**：https://www.youtube.com/watch?v=SKNNIrytvpw

## 概要

「プログラミング初心者はPythonから始めるべき」という常識に異を唱える動画。むしろ**初心者こそRustを学ぶべき**という提案を5章構成で詳しく解説。

## 5つの主要なポイント

### 第1章：なぜ「初心者はPythonから」が常識か

Pythonは簡単で楽しいが、裏側で自動管理される仕組みを隠してしまう問題がある：

- メモリ管理、型の安全性、並行処理の安全性が全て自動化される
- 1～2年、気にしない状態で過ごすと「気にできない状態」になってしまう
- システムが大きくなってから「なぜメモリが足りないのか」と直面しても、メモリの概念を知らないため原因が分からない
- オートマ車で運転を覚えた人がエンジンの仕組みを知らないのと同じ

**後付け学習の辛さ**：パフォーマンス問題やセキュリティ穴に直面してから、低レイヤーの知識を学び直すのは非常に大変。最初から知っている人との差が埋まらない。

### 第2章：Rustが「厳しい先生」である理由

**所有権と借用（Ownership & Borrowing）**
- データは1人でしか借りられない（図書館の貸本制度のような仕組み）
- 借りたら返す約束をし、同時に複数が書き換えることはできない

**Rustのコンパイラの特徴**
- このルールに違反したコードを一切通さない
- 実行前にすべてチェック（Python/JavaScriptは実行時に発覚）
- ただし「何が間違っているか」だけでなく「どう直せばいいか」も教えてくれる
- エラーメッセージが親切で、ヒント付き

**愛される厳しさ**：Stack Overflow 2025年調査で、Rustは最も愛されている言語。使用経験者の72%が「また使いたい」と回答。高い学習コストなのに満足度が最も高い。

### 第3章：厳しい先生に鍛えられると何が身につくか

**1. メモリの理解**
- 所有権と借用を学ぶことで、データがメモリでどう管理されるかを嫌でも理解する
- Pythonだけではまず触れない概念
- 他の言語を書く時も応用できる

**2. 安全なコードを書く感覚**
- 変数を誰が書き換えているか、並行処理で競合しないかを常に意識する癖がつく
- Java、Go、TypeScriptなど、言語を超えた直感が育つ

**3. エラーに強くなる**
- コンパイラに何百回も怒られた経験がある人はエラーメッセージを読む習慣がつく
- エラー = パニック ではなく、エラー = 友達 という感覚になる
- プログラミング界で最も親切とも言われるRustのエラーメッセージは最高の教材

### 第4章：世界がRustを選び始めている

**採用状況**
- 開発者数：226万人以上が使用、70万人以上が主要言語として採用
- 商用利用急増：2021～2024年で68%以上増加

**大企業の採用**
- **Microsoft**：新規プロジェクトではC++ではなくRustを使うべき（公式発言）
- **Google**：Androidに採用 → メモリ関連バグが劇的に減少
- **AWS**：Firecracker（マイクロVM）を全てRustで構築

**基盤への組み込み**
- **Linuxカーネル**：2021年からRust組み込み開始（50年以上C言語だけだった歴史を初めて変えた）
- **WebAssembly**：有力言語の1つ

**歴史的背景**：Rustは2015年版1.0リリース。11年後にLinuxカーネルに入るという、プログラミング言語史上かなり例外的な速度での採用。

### 第5章：最初に楽な道 vs 最初にきつい道

**Pythonから始めた場合**
- 最初は楽で楽しい
- 壁に直面（パフォーマンス、メモリリーク、本番バグ）
- 後付けで低レイヤーを学ぶはめになる
- 「気にしない癖」がついているため、切り替え時に精神的コストが大きい

**Rustから始めた場合**
- 最初から「正しいコード」を叩き込まれる
- その後Python/JavaScriptを学ぶと「裏側でこうなってるんだ」と理解できる
- 筋トレで重い荷物に慣れた後の普通の荷物は軽く感じる感覚
- 最初にきつい負荷をかけた人が最終的に楽に動ける

**推奨学習方式**：Rust単体ではなく、Rustを学びつつ並行してPythonで楽しいものを作る（筋トレだけでなく実践も混ぜる）。挫折を防ぎながら根本的なスキルを身につける。

## 重要なポイント

- **「初心者に Rust は無理」は誤解**：Rust が難しいのではなく、プログラミングの根本が難しいだけ。Rust はそれを隠さず、コンパイル時に教えてくれる。
- **Rust が親切な理由**：厳しいエラーメッセージには解法が含まれている。
- **学習リソースは充実**：公式ドキュメント「The Rust Programming Language」は日本語翻訳あり。世界中で「最高の公式ドキュメント」と評価されている。

## 結論

> **「最初にきつい道を選んだ人が、最終的に一番遠くまで行ける」**

プログラミング初心者のキャリアを長期的に考えると、Rust は有力な選択肢。避けて通れない低レイヤーの知識は、どうせ身につける必要があるなら、最初に学んだ方が得。
