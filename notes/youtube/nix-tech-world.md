# Nixとは何か？エンジニアの間で流行している理由

https://www.youtube.com/watch?v=urpzO1r1Y1c

**出演者：** ryoppippi（Nixエキスパート）、たけてぃ、マグロ（NixOSユーザー）、市川（ホスト）

## Nixとは

Nixは**パッケージマネージャー**。Homebrew・apt・npm など複数のパッケージマネージャーに散らばっていた依存関係を**一元管理**できるのが最大の特徴。

世界最大のパッケージレジストリ「nixpkgs」を持ち、コマンドラインツールからランタイム（Node、Python等）まで幅広くカバーしている。

## 「おま環」問題を解決する

**「おま環」**（お前の環境だけだろ）＝「自分のマシンでは動くのに…」という再現性のない環境の問題。

`flake.nix` というファイルにプロジェクトの依存関係を宣言的に定義し、`flake.lock` でコミットハッシュまで固定することで：

- チームメンバー全員が同じバージョンを使える
- CI（GitHub Actions）でも同じ環境を再現できる
- セットアップ手順書が不要になる（`nix develop` 一発で環境構築完了）

## グローバル環境を汚さない

- Nixはすべてのパッケージを `/nix/store` に格納し、シンボリックリンクを貼るだけ
- プロジェクトごとに異なるバージョンのツール（Git、Node等）を共存させられる
- ryoppippiはシステムグローバルにNode/PHPを入れず、各プロジェクトにのみツールを定義している

## デメリット

- プロジェクト初回立ち上げ時の設定コピー作業
- プロジェクトごとに異なるバージョンを使うと、その分ディスク容量を消費する

## その他の活用方法

- **devenv / process-compose**：PostgreSQL等のプロセスをホスト上で直接管理。Docker Compose の代替として使えるが、あくまで開発環境向け（コンテナ隔離なし）
- **ドットファイルズ管理**（Home Manager）：Macのキーバインド、マウス感度、ドックの設定まで宣言的に管理
- **NixOS**：OSの設定すべてをNixで管理。新しいマシンを買ってもリポジトリをcloneして一発で同じ環境を再現できる

## AI/LLM時代との相性

「LLMにとって気持ちいい環境は人間にも気持ちいい」。環境構築で迷わない = LLMも迷わない。Claude Code に渡すと `flake.nix` を生成してくれるため、AI 時代の開発環境構築としても最適。

## 始め方（段階的に）

1. **プロジェクト単位**から始める（`flake.nix` で依存関係を管理）
2. **ユーザー環境**に広げる（Home Manager でドットファイルズ管理）
3. **NixOS**（OS 全体の管理、上級者向け）

**すぐやめられるのも強み**：Nixはシンボリックリンクを貼るだけなので、やめたければ Nix ストアを消してシンボリックリンクを外すだけ。既存の環境には一切干渉しない。
